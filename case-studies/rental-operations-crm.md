# Real Estate Rental Operations CRM Automation

## Problem

A rental/property operation needed to manage leads, units, rental agreements, service requests, serial numbers, related-record updates, and banking refreshes across CRM and external systems.

## Solution

Built Make scenarios that connected email/webhook triggers, Fireberry CRM records, AI-assisted extraction, HTTP APIs, and operational update workflows.

## Workflow Capabilities

- Multi-source rental lead inbox parsing.
- AI-assisted extraction for unstructured lead emails.
- Unit/publication lookup and association.
- Rental agreement creation and linked-record updates.
- Serial number generation across rental entities.
- Service request creation from email and attachments.
- Related-record updates when unit names changed.
- Bank transaction refresh automation.

## Systems

Make, Fireberry CRM, Outlook/email, HTTP APIs, AI extraction modules, Google Vision-style OCR, banking API refresh patterns.

## Portfolio Value

This is the strongest Make/Fireberry case study because it shows full business-system architecture, not isolated automation. It connects lead intake, operational entities, financial refreshes, and service workflows.
